# VChatSyncCenter 使用说明（通俗版）

这份文档讲怎么使用 `VChatSyncCenter`。它是“同步中心”，可以理解成所有设备共同对账的服务器：桌面端和未来移动端都把自己的变更交给它，再从它这里拉取其他设备的变更。

## 一句话理解

- `VChatSyncCenter` 放在 **VCPToolBox** 里运行。
- 它保存中心数据库、附件、设备列表、变更记录和冲突记录。
- 桌面端的 `VChatSyncAdapter` 要连接到它。
- 它不直接改 `VCPChat/AppData`，桌面本地文件由 Adapter 负责写回。

## 你需要先准备什么

1. 一台作为同步中心的机器，运行 VCPToolBox。
2. VCPToolBox 已经能正常启动，并能访问插件 API。
3. 本插件目录存在：

```text
G:\VCP\VCPToolBox\Plugin\VChatSyncCenter
```

4. 准备一串同步密钥：

```text
请改成你自己的长随机字符串，不要长期使用 change-me、123456 或简单数字
```

同步密钥要同时配置在：

- VCPToolBox 侧 `VChatSyncCenter`。
- 每台 VCPChat 桌面端的 `VChatSyncAdapter/config.env`。

## 第 1 步：启用中心插件

`plugin-manifest.json` 里声明了本插件是 service 插件，并且会挂载 API：

```json
{
  "name": "VChatSyncCenter",
  "pluginType": "service",
  "hasApiRoutes": true
}
```

正常情况下，VCPToolBox 启动插件后，会把接口挂到：

```text
http://<VCPToolBox主机>:<端口>/api/plugins/VChatSyncCenter
```

例如你的 VCPToolBox 在 `100.100.50.10:6005`，那么中心地址就是：

```text
http://100.100.50.10:6005/api/plugins/VChatSyncCenter
```

## 第 2 步：配置中心插件

中心插件关键配置：

| 配置项 | 建议 | 说明 |
| --- | --- | --- |
| `VCHAT_SYNC_ENABLED` | `true` | 启用同步中心 |
| `VCHAT_SYNC_KEY` | 改成高强度随机字符串 | 所有同步请求都靠它鉴权 |
| `VCHAT_SYNC_REQUIRE_DEVICE_BINDING` | `true` | 要求设备注册后再同步 |
| `VCHAT_SYNC_WS_ENABLED` | 可先 `false` | WebSocket 只做通知，REST 轮询也能同步 |
| `VCHAT_DB_PATH` | 默认即可 | 中心 SQLite 数据库 |
| `VCHAT_ATTACHMENT_DIR` | 默认即可 | 附件存储目录 |
| `VCHAT_BACKUP_DIR` | 默认即可 | 备份目录 |

默认路径：

```text
数据库：./Plugin/VChatSyncCenter/data/vchat_data.db
附件：  ./Plugin/VChatSyncCenter/data/vchat_attachments
备份：  ./Plugin/VChatSyncCenter/data/backups
```

> 如果要把数据放到代码目录之外，可以把这些路径改到独立数据盘。个人使用先用默认路径也可以。

## 第 3 步：重启 VCPToolBox

修改配置后，重启 VCPToolBox，让插件重新加载。

PowerShell 示例：

```powershell
$Center = "http://100.100.50.10:6005/api/plugins/VChatSyncCenter"
$Key = "你的同步密钥"
$Headers = @{ Authorization = "Bearer $Key" }

Invoke-RestMethod -Uri "$Center/status" -Headers $Headers -Method Get
```

正常会看到类似：

```json
{
  "ok": true,
  "enabled": true,
  "latest_seq": 0,
  "db": {
    "ready": true,
    "integrity": "ok"
  }
}
```

重点看：

- `ok: true`：中心可用。
- `enabled: true`：同步中心已启用。
- `latest_seq`：中心当前最新变更序号。
- `integrity: ok`：数据库完整性正常。

## 常用命令

下面用 PowerShell 5.x 写法。

### 1. 检查中心状态

```powershell
Invoke-RestMethod -Uri "$Center/status" -Headers $Headers -Method Get
```

适合：刚启动中心、Adapter 连不上中心、想看数据库和备份状态时。

### 2. 注册设备

通常 Adapter 会自动注册设备。你也可以手动测试：

```powershell
$Body = @{
  device_id = "pc-home"
  name = "家里的电脑"
  platform = "win32"
} | ConvertTo-Json

Invoke-RestMethod -Uri "$Center/devices/register" -Headers $Headers -Method Post -ContentType "application/json" -Body $Body
```

### 3. 查看变更列表

```powershell
Invoke-RestMethod -Uri "$Center/changes?after_seq=0&limit=20" -Headers $Headers -Method Get
```

返回里会有：

- `latest_seq`：中心当前最新 seq。
- `events`：变更事件列表。
- `has_more`：是否还有更多。
- `next_after_seq`：下一次继续拉取的位置。

### 4. 查看冲突记录

```powershell
Invoke-RestMethod -Uri "$Center/conflicts?limit=50" -Headers $Headers -Method Get
```

适合：多台设备同时改同一条消息后，或某些编辑结果不符合预期时。

### 5. 创建数据库备份

```powershell
$Body = @{ reason = "manual-before-sync-test" } | ConvertTo-Json
Invoke-RestMethod -Uri "$Center/backup/create" -Headers $Headers -Method Post -ContentType "application/json" -Body $Body
```

建议在第一次接入、大量导入、merge 前执行。

### 6. 查看备份列表

```powershell
Invoke-RestMethod -Uri "$Center/backup/list" -Headers $Headers -Method Get
```

### 7. 校验备份

```powershell
$Body = @{ name = "备份文件名" } | ConvertTo-Json
Invoke-RestMethod -Uri "$Center/backup/verify" -Headers $Headers -Method Post -ContentType "application/json" -Body $Body
```

### 8. 恢复备份（谨慎）

恢复会影响中心数据库，是高风险操作。建议先确认当前所有客户端暂停同步，再恢复。

```powershell
$Body = @{ name = "备份文件名" } | ConvertTo-Json
Invoke-RestMethod -Uri "$Center/backup/restore" -Headers $Headers -Method Post -ContentType "application/json" -Body $Body
```

## 同步完整流程：第一台电脑怎么接入

1. 启动 VCPToolBox，并启用 `VChatSyncCenter`。
2. 配好 `VCHAT_SYNC_KEY`。
3. 用 `/status` 确认中心正常。
4. 到第一台 VCPChat 桌面端配置 `VChatSyncAdapter/config.env`。
5. 第一台桌面端执行 `bootstrap_primary`，把本机 AppData 作为初始基线导入中心。
6. 等 Adapter 状态正常后，日常聊天就会自动同步。

中心侧可以看：

```powershell
Invoke-RestMethod -Uri "$Center/status" -Headers $Headers -Method Get
```

`latest_seq` 应该会随着桌面端导入和聊天变化增长。

## 同步完整流程：第二台电脑怎么接入

1. 第二台电脑也安装/运行 VCPChat 和 `VChatSyncAdapter`。
2. `config.env` 指向同一个中心地址。
3. `VCHAT_SYNC_KEY` 与中心完全一致。
4. `VCHAT_DEVICE_ID` 要与第一台不同，例如：

```text
第一台：pc-home
第二台：pc-office
```

5. 第二台通常使用 `join_existing`，从中心拉基线并投影到本机 AppData。
6. 之后两台电脑都通过中心同步。

## 配置同步和 profile 隔离

配置同步分两类：

| profile | 用途 |
| --- | --- |
| `bootstrap` | 首次导入/导出完整安全基线 |
| `runtime` | 日常运行中的局部字段同步 |

Center 数据库里 `config_entities` 使用：

```text
schema + entity_id + profile
```

作为逻辑主键。所以同一个 Agent 的 bootstrap 配置和 runtime 局部配置会分开保存，不会互相覆盖。

Bootstrap 导出只导出 `bootstrap` profile。这样新设备加入时拿到的是完整安全基线，而不是某台旧设备发来的局部 runtime 配置。

## 和 Adapter 的分工

```text
VChatSyncCenter：负责“存”和“分发”
VChatSyncAdapter：负责“读写本机 AppData”和“自动同步”
```

中心插件常看：

- `/status`：中心是否活着。
- `/changes`：有没有变更。
- `/conflicts`：有没有冲突。
- `/backup/*`：备份和恢复。

Adapter 常看：

- `/api/vchat-sync-adapter/status`：本机是否正常扫描、排队、拉取、投影。
- `/api/vchat-sync-adapter/bootstrap/*`：首次接入方式。

## 常见问题

### 1. 返回 401 或 authorization failed

通常是同步密钥不一致。检查：

- 中心 `VCHAT_SYNC_KEY`。
- Adapter `config.env` 里的 `VCHAT_SYNC_KEY`。
- 命令里的 `$Key`。

三处必须完全一样。

### 2. `/status` 正常，但 Adapter 不同步

先去 Adapter 侧看：

```powershell
$Adapter = "http://127.0.0.1:你的VCPChat端口/api/vchat-sync-adapter"
Invoke-RestMethod -Uri "$Adapter/status" -Headers $Headers -Method Get
```

重点看：

- `enabled` 是否为 true。
- `mode` 是否不是 `uninitialized`。
- queue 是否堆积。
- pull 是否有错误。

### 3. WebSocket 关闭会不会不能同步

不会。WebSocket 只做 latest_seq 通知；REST `/changes` 才是可靠同步通道。关闭 WebSocket 时，Adapter 会靠轮询同步，只是延迟可能变成十几秒。

### 4. 可以直接改中心数据库吗

不建议。直接改数据库可能产生 Adapter 无法投影的事件，导致同步卡住。请优先通过 API 和正常同步流程处理。

### 5. migration 会不会因为列已存在启动失败？

当前实现已处理这类兼容问题。migration 遇到重复列会跳过对应语句，并继续完成后续表结构迁移。旧的 `runtime:xxx` / `manual:xxx` 配置记录也会迁移成真实 profile 行。

## 安全提醒

- 不要把 `VCHAT_SYNC_KEY` 发给别人。
- 不要长期使用 `change-me`、`123456`、`258456` 这类简单密钥。
- 备份恢复前先手动创建一份新备份。
- 重要数据首次接入前，建议先完整备份 `VCPChat/AppData` 和中心数据库。
- 恢复中心备份前建议暂停所有 Adapter，恢复完成后再逐台恢复同步。
